---
name: new-chapter
description: techbook-template プロジェクトに新しい章ファイルを作成する。ユーザーから章番号・タイトル・サブタイトル・2段組設定を聞き、docs/ ディレクトリに Markdown ファイルを生成する。
---

ユーザーに以下を確認してから新しい章ファイルを作成してください:

1. **章番号** - ファイル名の番号部分（例: `03`）
2. **ファイル名の内容部分** - 英数字・ハイフン形式（例: `advanced-settings`）
3. **章タイトル（日本語）** - H1 ヘッダーに使用
4. **サブタイトル（任意）** - H1 直後の1行
5. **2段組にするか** - `columns: false`（1段組）にする場合は明示する（デフォルトは2段組）

確認後、`docs/{番号}-{内容}.md` に以下の雛形でファイルを作成してください:

```markdown
---
description: （任意：章の説明）
columns: false
---
# {章タイトル}

{サブタイトル}

## はじめに

ここから本文を書き始めます。
```

**注意点:**
- `columns: false` はユーザーが1段組を希望した場合のみ記述する（2段組の場合は frontmatter ごと省略可）
- ファイルを作成したら、既存の章ファイル一覧（`docs/` の内容）を確認して番号の重複がないかチェックする
- 作成後、`npm run dev` でプレビューを確認するよう案内する
